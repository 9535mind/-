# 🎯 카카오 로그인 설정 - 사용자 실행 가이드

> **대상**: Mind스토리 LMS 관리자  
> **목적**: 카카오 개발자 콘솔에서 직접 설정하기  
> **소요 시간**: 약 10분  
> **난이도**: ⭐⭐☆☆☆ (쉬움)

---

## 📋 시작하기 전에

### 준비물
- ✅ 카카오 개발자 계정 (카카오톡 계정으로 로그인 가능)
- ✅ PC 또는 노트북 (모바일은 불편함)
- ✅ 약 10분의 시간
- ✅ 인터넷 연결

### 주의사항
- 각 단계마다 **"저장"** 버튼을 꼭 클릭하세요
- 저장 후 **브라우저 새로고침(F5)**으로 확인하세요
- 복사/붙여넣기 시 **앞뒤 공백**을 조심하세요

## 🚀 Step 1: 카카오 개발자 콘솔 접속

### 소요 시간: 1분

### 1-1. 웹사이트 접속
```
https://developers.kakao.com
```

### 1-2. 로그인
1. 우측 상단 **"로그인"** 클릭
2. 카카오톡 계정으로 로그인
3. (처음이면) 약관 동의

### 1-3. 앱 찾기
1. 상단 메뉴에서 **"내 애플리케이션"** 클릭
2. 앱 목록에서 **"Mind스토리 LMS"** 클릭

📸 **화면 예시:**
```
┌────────────────────────────────────┐
│  내 애플리케이션                   │
├────────────────────────────────────┤
│  [Mind스토리 LMS] ← 이것 클릭     │
│  앱 키: 1360027                    │
└────────────────────────────────────┘
```

✅ **확인:** 앱 이름 "Mind스토리 LMS"가 상단에 보임

## 📝 Step 2: Redirect URI 등록 ⭐⭐⭐⭐⭐

### 소요 시간: 2분
### 중요도: 최상 (가장 중요한 단계!)

### 2-1. 메뉴 이동
1. 왼쪽 사이드바 찾기
2. **"제품 설정"** 클릭
3. **"카카오 로그인"** 클릭

### 2-2. Redirect URI 섹션 찾기
- 페이지를 아래로 스크롤
- **"Redirect URI"** 제목 찾기

### 2-3. 기존 URI 확인 및 정리
기존에 등록된 URI가 있다면:
1. 각 URI 오른쪽의 **"×"** 버튼 클릭
2. **모두 삭제**

⚠️ **왜 삭제하나요?**  
→ 불일치하는 URI가 있으면 로그인이 작동하지 않습니다

### 2-4. 새 URI 등록
1. **"+ Redirect URI 등록"** 버튼 클릭
2. 팝업창이 나타남
3. 아래 주소를 **정확히 복사**해서 붙여넣기:

```
https://3000-ieu1ambselnpjf2cme9se-c81df28e.sandbox.novita.ai/api/auth/kakao/callback
```

### 2-5. 중요 확인 사항 (반드시!)

복사/붙여넣기 후 다음을 확인하세요:

- [ ] `https`로 시작 (http 아님!)
- [ ] 마지막에 슬래시(`/`) **없음**
- [ ] 앞에 공백 **없음**
- [ ] 뒤에 공백이나 줄바꿈 **없음**
- [ ] `/api/auth/kakao/callback` 경로 포함

❌ **잘못된 예시:**
```
https://3000-ieu1ambselnpjf2cme9se-c81df28e.sandbox.novita.ai/api/auth/kakao/callback/  (끝에 슬래시)
http://3000-ieu1ambselnpjf2cme9se-c81df28e.sandbox.novita.ai/api/auth/kakao/callback   (http)
 https://3000-ieu1ambselnpjf2cme9se-c81df28e.sandbox.novita.ai/api/auth/kakao/callback (앞에 공백)
```

✅ **올바른 예시:**
```
https://3000-ieu1ambselnpjf2cme9se-c81df28e.sandbox.novita.ai/api/auth/kakao/callback
```

### 2-6. 저장
1. **"확인"** 버튼 클릭 (팝업창)
2. **"저장"** 버튼 클릭 (페이지 하단)

### 2-7. 저장 확인
1. 브라우저 새로고침 (F5 키)
2. Redirect URI 목록에 방금 등록한 URI가 보이는지 확인

📸 **완료 화면:**
```
┌────────────────────────────────────────────────────┐
│ Redirect URI                                       │
├────────────────────────────────────────────────────┤
│ https://3000-ieu1ambselnpjf2cme9se-...        [×] │
└────────────────────────────────────────────────────┘
```

✅ **확인:** URI가 목록에 표시됨

"사이트 도메인" 입력란에 아래 주소 **정확히** 입력:


### 3-4. 중요 확인 사항

- [ ] `https` 포함
- [ ] 마지막에 슬래시(`/`) **없음**
- [ ] 경로 없음 (도메인만!)
- [ ] `/api/auth/kakao/callback` 같은 경로 **절대 붙이지 마세요**

❌ **잘못된 예시:**
```
https://3000-ieu1ambselnpjf2cme9se-c81df28e.sandbox.novita.ai/                          (끝에 슬래시)
https://3000-ieu1ambselnpjf2cme9se-c81df28e.sandbox.novita.ai/api/auth/kakao/callback  (경로 포함)

```

✅ **올바른 예시:**
```
https://3000-ieu1ambselnpjf2cme9se-c81df28e.sandbox.novita.ai
```

### 3-5. 저장
1. **"확인"** 버튼 클릭 (팝업창)
2. **"저장"** 버튼 클릭 (페이지 하단)

### 3-6. 저장 확인
1. 브라우저 새로고침 (F5)
2. "Web" 플랫폼이 목록에 나타나는지 확인

📸 **완료 화면:**

## 🔓 Step 4: 카카오 로그인 활성화

### 소요 시간: 1분
### 중요도: 높음

### 4-1. 메뉴 이동
1. 왼쪽 사이드바에서 **"제품 설정"** 클릭
2. **"카카오 로그인"** 클릭

### 4-2. 활성화 설정
페이지 상단에서 **"카카오 로그인 활성화"** 토글 찾기

현재 상태:
- **OFF (회색)** → 클릭해서 **ON (파란색)**으로 변경
- **ON (파란색)** → 이미 활성화됨, 그대로 두기

📸 **OFF 상태:**
```
카카오 로그인 활성화  [───●] OFF  (회색)
```

📸 **ON 상태:**
```
카카오 로그인 활성화  [●───] ON  (파란색)
```

### 4-3. 저장
- **"저장"** 버튼 클릭 (페이지 하단)

✅ **확인:** 토글이 파란색 ON 상태

## 👥 Step 5: 테스트 사용자 등록 (조건부)

### 소요 시간: 2분
### 중요도: 중간 (조건에 따라 필요)

⚠️ **먼저 확인하세요!**

### 5-1. 앱 상태 확인
1. 왼쪽 사이드바 > **"앱 설정"** > **"일반"**
2. 페이지 상단에서 **"앱 상태"** 확인

**앱 상태가 "개발 중"** → 아래 단계 진행 ✅  
**앱 상태가 "서비스 중"** → 이 단계 건너뛰기 ⏭️

### 5-2. 테스트 사용자 관리 (개발 중인 경우만)
1. 같은 페이지에서 아래로 스크롤
2. **"테스트 사용자 관리"** 섹션 찾기

### 5-3. 카카오계정 이메일 입력
1. 본인의 **카카오계정 이메일** 입력
2. (확인 방법: 카카오톡 > 더보기 > 설정 > 계정 관리)

### 5-4. 저장
- **"확인"** 버튼 클릭

📸 **완료 화면:**
```
┌────────────────────────────────────────────────────┐
│ 테스트 사용자 관리                                 │
├────────────────────────────────────────────────────┤
│ your-email@kakao.com                          [×]  │
└────────────────────────────────────────────────────┘
```

✅ **확인:** 본인 이메일이 목록에 보임

## 🧪 Step 7: 테스트

브라우저 **시크릿 모드** 열기:
- Chrome/Edge: `Ctrl + Shift + N`
- Firefox: `Ctrl + Shift + P`
- Safari: `Cmd + Shift + N`

⚠️ **왜 시크릿 모드인가요?**  
→ 기존 로그인 정보가 테스트를 방해하지 않도록

### 7-2. 회원가입 페이지 접속
시크릿 모드에서 아래 주소 입력:
```
https://3000-ieu1ambselnpjf2cme9se-c81df28e.sandbox.novita.ai/register
```

### 7-3. 카카오 로그인 시도
1. **"카카오로 1초 만에 시작하기"** 버튼 클릭

3. 카카오 계정 입력 (또는 자동 로그인)
4. 동의 항목 확인 (있을 경우)

### 7-4. 결과 확인

```
3. 우측 상단에 본인 이름 표시
```

🎉 **축하합니다! 설정이 완료되었습니다!**

## 🚨 문제 해결

### 문제 1: Redirect URI 불일치 에러

#### 증상
```
에러 코드: KOE006
메시지: redirect_uri mismatch
```

#### 원인
Redirect URI가 정확히 일치하지 않음

#### 해결 방법
1. **Step 2**로 돌아가기
2. 기존 Redirect URI 삭제
3. 아래 URI를 **정확히** 복사해서 붙여넣기:
   ```
   https://3000-ieu1ambselnpjf2cme9se-c81df28e.sandbox.novita.ai/api/auth/kakao/callback
   ```
4. 앞뒤 공백, 마지막 슬래시 확인
5. 저장 후 브라우저 새로고침
6. **Step 7**로 돌아가서 다시 테스트

---

### 문제 2: 서비스 설정 오류

#### 증상
```
에러 코드: KOE101 또는 KOE009
메시지: 서비스 설정이 올바르지 않습니다
```

#### 원인
Web 플랫폼 도메인이 등록되지 않았거나 잘못됨

#### 해결 방법
1. **Step 3**으로 돌아가기
2. Web 플랫폼 설정 확인
3. 경로 없이 도메인만 입력했는지 확인:
   ```
   https://3000-ieu1ambselnpjf2cme9se-c81df28e.sandbox.novita.ai
   ```
4. 저장 후 다시 테스트

---

### 문제 3: 흰 화면 또는 무반응

#### 증상
- 버튼 클릭 후 아무 반응 없음
- 흰 화면만 보임

#### 해결 방법
1. 브라우저 개발자 도구 열기 (F12)
2. "Console" 탭 클릭
3. 빨간색 에러 메시지 확인
4. 에러 메시지 스크린샷
5. 시크릿 모드에서 다시 시도
6. 여전히 안 되면 스크린샷 공유

---

### 문제 4: 테스트 사용자가 아닙니다

#### 증상
```
메시지: 앱 관리자나 테스트 사용자만 이용할 수 있습니다
```

#### 원인
앱이 "개발 중" 상태인데 테스트 사용자로 등록되지 않음

#### 해결 방법
1. **Step 5**로 돌아가기
2. 앱 설정 > 일반 > 앱 상태 확인
3. "개발 중"이면:
   - 테스트 사용자 관리에서 본인 이메일 추가
4. 다시 테스트

---

### 문제 5: 인증 코드 없음

#### 증상
로그인 후 에러 페이지:
```
⚠️ 카카오 로그인 실패
인증 코드를 받지 못했습니다.
```

#### 원인
위의 4가지 설정 중 하나 이상이 잘못됨

#### 해결 방법
1. **Step 6** 체크리스트로 돌아가기
2. 모든 항목 다시 확인
3. 특히 **Redirect URI**와 **Web 플랫폼** 재확인
4. 브라우저 캐시 삭제
5. 시크릿 모드로 다시 테스트

## 📸 문제 지속 시 필요한 정보

아래 4가지 스크린샷을 찍어서 공유해주세요:

### 1️⃣ Redirect URI 설정 화면
- **경로**: 제품 설정 > 카카오 로그인
- **찍을 부분**: Redirect URI 목록 전체
- **확인 사항**: 등록된 URI가 선명히 보이도록

### 2️⃣ Web 플랫폼 설정 화면
- **경로**: 앱 설정 > 플랫폼
- **찍을 부분**: 등록된 플랫폼 목록
- **확인 사항**: "Web" 플랫폼과 도메인이 보이도록

### 3️⃣ 카카오 로그인 활성화 상태
- **경로**: 제품 설정 > 카카오 로그인
- **찍을 부분**: 페이지 상단
- **확인 사항**: "카카오 로그인 활성화" 토글 상태 (ON/OFF)

### 4️⃣ 앱 상태 및 테스트 사용자
- **경로**: 앱 설정 > 일반
- **찍을 부분**: 앱 상태, 테스트 사용자 관리
- **확인 사항**: "개발 중"/"서비스 중" 상태, 테스트 사용자 목록
