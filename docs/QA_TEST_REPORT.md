#### B. 환불 로직 테스트

**환불 규정**:
1. **진도 0% + 7일 이내**: 100% 환불
2. **진도 50% 미만**: 50% 환불
3. **진도 50% 이상**: 환불 불가

**테스트 시나리오**:

| 시나리오 | 진도율 | 결제일 | 환불액 | 상태 | 결과 |
|---------|--------|--------|--------|------|------|
| 1. 수강 전 환불 | 0% | 3일 전 | 100% | ✅ 통과 | 전액 환불 정상 |
| 2. 7일 경과 환불 | 0% | 10일 전 | 50% | ✅ 통과 | 50% 환불 정상 |
| 3. 일부 수강 환불 | 30% | 5일 전 | 50% | ✅ 통과 | 50% 환불 정상 |
| 4. 절반 이상 환불 | 60% | 2일 전 | 0% | ✅ 통과 | 환불 불가 에러 |

**환불 로직 코드 검증**:
```typescript
// src/routes/payments-v2.ts
// 환불 계산 로직
const daysSincePurchase = Math.floor(
  (Date.now() - new Date(payment.created_at).getTime()) / (1000 * 60 * 60 * 24)
)

if (progressRate === 0 && daysSincePurchase <= 7) {
  refundAmount = payment.amount // 100%
} else if (progressRate < 50) {
  refundAmount = Math.floor(payment.amount * 0.5) // 50%
} else {
  return c.json(errorResponse('진도율이 50% 이상이므로 환불할 수 없습니다.'), 400)
}
```

**✅ 환불 로직 정상 작동 확인**

### 4️⃣ 영상 진도율 조작 방지

#### A. 진도율 추적 시스템

**구현된 보안 기능**:
1. ✅ **5초마다 자동 저장**: 사용자가 수동으로 저장 불가
2. ✅ **건너뛰기 감지**: 시청 시간 vs 현재 시간 비교
3. ✅ **세션 기반 검증**: 수강권 있는 사용자만 시청
4. ✅ **Range 요청 제한**: 순차 시청 강제
5. ⚠️ **영상 워터마크**: 미구현

**진도율 저장 로직**:
```typescript
// public/static/js/video-player.js (커스텀 플레이어)
// 5초마다 진도율 자동 저장
setInterval(() => {
  const currentProgress = (video.currentTime / video.duration) * 100
  saveProgress(enrollmentId, lessonId, currentProgress, video.currentTime)
}, 5000)

// 건너뛰기 감지
video.addEventListener('seeked', () => {
  const jumpAmount = Math.abs(video.currentTime - lastSavedTime)
  if (jumpAmount > 10) { // 10초 이상 건너뛰기
    alert('영상을 건너뛸 수 없습니다.')
    video.currentTime = lastSavedTime
  }
})
```

#### B. 조작 방지 테스트

| 시나리오 | 조작 방법 | 상태 | 결과 |
|---------|----------|------|------|
| 1. 건너뛰기 | Seek bar 드래그 | ✅ 차단 | 10초 이상 점프 시 되돌림 |
| 2. 2배속 재생 | playbackRate 변경 | ⚠️ 미차단 | 브라우저 기본 기능 |
| 3. 개발자 도구 | currentTime 직접 수정 | ⚠️ 부분 차단 | Seeked 이벤트로 감지 |
