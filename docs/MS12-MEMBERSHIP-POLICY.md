# MS12 회원·요금제 정책 설계서 (초안)

**기준 문서**: 회원 정책·요금제·**화면(UI) 설계**의 단일 기준은 본 파일 `docs/MS12-MEMBERSHIP-POLICY.md` 로 유지한다. 구현 시 코드·카피는 여기서 벗어나지 않도록 한다.

**목적**: 회원 기반 회의록 플랫폼 전환 시 정책·제한·화면 흐름을 코드 수정 전에 고정한다.  
**범위**: MS12(`/app*`·`/api/ms12*` 등). **FOREST·mslms는 대상 외**(별도 코드·경로 수정 금지).  
**결제**: 초기에는 **외부 결제 연동 없음**. 사용자·조직에는 **`plan`(또는 `users.ms12_plan`) 필드만** 두고 `free` / `pro` / `org` 로 구분한다. 실제 과금·청구는 후속 단계에서 분리한다.

**문서 버전**: 초안 · 2026-04-22（§6 경로별 UI 보강）

## 1. 회원 정책 설계서

### 1.1 제품 포지션

- MS12는 **공개 무규칙 사용형이 아니라**, **회원(계정) 단위로 회의록을 적재·관리하는 플랫폼**으로 간다.
- **회의 생성·핵심 기록 기능**은 로그인 후에만 허용한다(비로그인은 소개·가입·로그인 유도).

### 1.2 계정 유형과 DB 표현

| 구분 | 의미 | DB `plan` 값 |
|------|------|----------------|
| 비로그인 | 세션 없음 또는 게스트만 | *(레코드 없음 또는 명시적 guest — 구현 시 선택)* |
| 무료 회원 | 회원가입 완료, 기본 한도 내 이용 | `free` |
| 유료 회원 | 긴 회의·확대 저장·AI·공유 등 상위 기능 | `pro` |
| 기관 회원 | 다 사용자·보관·관리자·계약 단위 | `org` |

- 필드명 예시: **`users.ms12_plan`** 또는 통합 **`plan`** 컬럼(실제 스키마는 마이그레이션 시 확정).
- 신규 가입 시 기본값은 **`free`**.
- **`pro` / `org`**는 초기에는 운영에서 수동 설정 또는 계약 후 반영 가능하게 두고, 결제 게이트웨이 연동은 후속.

### 1.3 회원별 정책 요약

#### 무료 회원 (`free`)

- **회원가입 필수**(이메일·OAuth 등 구현 방식은 인증 설계와 함께 확정).
- **회의 생성 가능**(로그인 후).
- **시간·저장 개수 제한** 적용(구체 수치는 §2 표에서 관리).
- **기본 전사·기본 저장**까지 허용(상위 전사 길이·클라우드 STT 등은 제한 가능).

#### 유료 회원 (`pro`)

- **무료 한도 상향**: 더 긴 회의, 저장 건수 확대 등.
- **AI 요약·보고서 성격 기능**, **공유·대외 기능**(링크 공유 범위 등) 사용 허용 범위 확대.

#### 기관 회원 (`org`)

- **여러 사용자(좌석·조직 단위)** 관리.
- **회의 기록 보관** 정책(보존 기간·저장소 구분 등은 운영 규칙과 함께 확정).
- **관리자 대시보드** 접근.
- **결제·계약**은 비즈니스 프로세스로 두고, 시스템상으로는 `plan = org` 및 조직 메타데이터(후속 스키마)로 표현.

### 1.4 비로그인 정책

- **`/app`**: 서비스 소개 + **로그인·회원가입** 진입점만 제공.
- **회의 시작·회의 생성**은 **로그인 후**만 가능(현재 코드와 불일치 시, 구현 단계에서 라우트·버튼 게이팅으로 맞춤).

### 1.5 결제와 플랜의 분리

- 초기 배포: **stripe 등 결제 미연동**.
- DB에는 **`plan`(free/pro/org)** 만 유지하고, **`pro`/`org` 승격**은 관리 도구·수동·후속 결제 웹훅으로 연결.
- 과금 단위·환불·세금계산서 등은 별도 **결제 설계서**로 분리(PAYMENT 연동 문서 후속 갱신).

## 3. 비로그인 / 무료 / 유료 / 기관 회원별 화면 흐름

### 3.1 공통 원칙

- **진입점**: 비로그인 사용자는 **`/app`** 에서 마케팅·기능 소개와 **로그인·회원가입**만 노출하고, 회의 관련 깊은 경로는 로그인 후 열리도록 한다.
- **플랜 게이팅**: 한도 초과·유료 기능 클릭 시 → 업그레이드 안내(결제 연동 전에는 **문의 또는 준비 중** 카피).
- **경로별로 무엇을 보여줄지**(레이아웃·CTA·잠금)는 **§6** 에서 규정한다.

### 3.2 비로그인

```
/app (랜딩)
  → 서비스 소개
  → [로그인] → 기존 OAuth/이메일 로그인 플로우(구현 확정 시)
  → [회원가입] → 가입 폼(또는 OAuth 후 프로필 완료)
  → 회의 시작 버튼은 비활성 또는 클릭 시 로그인 유도 모달
```

- **`/app/meeting`, `/app/room/:id` 등**: 리다이렉트 또는 “로그인이 필요합니다” 중 선택(구현 시 단일 규칙으로 통일).

### 3.3 무료 회원 (`free`)

```
로그인 성공
  → /app 또는 회의 허브(/app/meeting 등)
  → 회의 생성 가능
  → 한도 내 전사·저장·기본 기능
  → AI·공유 등 유료 기능 진입 시 → pro 안내(TBD 카피)
```

### 3.4 유료 회원 (`pro`)

```
로그인 성공 (plan = pro)
  → 무료와 동일 네비게이션
  → 길이·저장·AI·공유 허용 범위 확대
  → 한도 도달 시 추가 과금 또는 안내(후속 결제 설계와 연결)
```

### 3.5 기관 회원 (`org`)

```
로그인 성공 (plan = org)
  → 기관 관리자: 사용자 초대·역할·보관 정책·대시보드
  → 일반 소속 사용자: org 정책에 따른 한도와 메뉴
```

- 조직 단위 네비게이션(예: `/app/org/...`)은 **스키마·라우트 설계 후속**으로 두어도 된다.

## 4. 구현 순서 제안 (참고 · 코드 작업 시)

1. 인증 필수화 및 **`plan` 필드** 마이그레이션·기본값 `free`.
2. 서버 API에서 회의 생성·저장·AI 호출 시 **플랜·한도 검증**.
3. `/app` 랜딩을 비로그인 소개형으로 조정, 회의 버튼 게이팅.
4. 결제 연동은 **`plan` 업데이트 소스**만 교체 가능하게 어댑터 분리.

## 6. 회원 상태별 화면(UI) 설계 — 경로별 정의

아래는 **구현 전 UI 설계 기준**이다. 실제 라우트 가드·리다이렉트 코드는 후속 작업에서 본 절과 일치시킨다.

### 6.1 비로그인

| 경로 | 화면에 보여야 하는 것 · 동작 |
|------|------------------------------|
| **`/app`** | 서비스 소개(가치 제안·회의록·보관 한 줄 요약 등). **주요 CTA**: `[로그인]` **`[회원가입]`**. 회의·전사 등은 **설명 텍스트만** 또는 스크린샷 수준. **회의 시작·새 회의** 버튼은 **표시하지 않거나**, 클릭 시 로그인 유도만(모달 또는 로그인 페이지로 이동). 상단/헤더에 **플랜 배지 없음**. |
| **`/app/meeting`** | **회의 허브 본문 대신** 접근 불가 처리 권장: **`/auth` 또는 `/app/login` 등 로그인 화면으로 리다이렉트**, 또는 빈 상태 + “회의 허브를 쓰려면 로그인하세요” + `[로그인]` `[회원가입]` 단일 플로우. (직접 URL 입력 방지 목적) |
| **`/app/room/:id`** | **회의실 미진입** 원칙: 로그인 페이지로 리다이렉트하거나, 동일하게 로그인 유도만. 초대 링크 UX가 필요하면 후속 정책으로 “로그인 후 원래 방으로 돌아오기(`next`)”만 허용. **비회원 게스트 회의실 표시는 하지 않음**(본 문서 범위에서는 회원 기반만 가정). |
| **`/app/record/:id`** | 기록 상세 **비표시**. 로그인 유도 화면 또는 로그인 후 **`next`** 로 복귀. |

### 6.2 무료 회원 (`plan = free`)

| 경로 | 화면에 보여야 하는 것 · 동작 |
|------|------------------------------|
| **`/app`** | 로그인 상태 표시(표시 이름). **플랜 배지 또는 라벨: 「무료」**(또는 Free). 필요 시 한 줄 안내: “무료 플랜 — 회의 시간·저장 개수에 한도가 있어요.” **CTA**: `[회의 허브로]` 또는 `[새 회의]` (실제 라우트명은 구현과 일치). 유료 기능 업셀은 **보조 배너**(§2 한도와 연동). |
| **`/app/meeting`** | **회의 허브 전체 기능** 중 무료에 허용된 범위: 회의 생성·빠른 시작·참여 링크·**최근 회의**(있을 경우)·보관함 진입 등. 상단에 **「무료」** 플랜 표시. **AI 확장·공유 확장·긴 회의** 등은 비활성·잠금 또는 클릭 시 **「Pro 안내」(결제 전에는 문의/준비중 카피)**. 한도에 근접 시 인라인 경고 배너(TBD 수치 연동). |
| **`/app/room/:id`** | 회의실 **기본 레이아웃**(메모·전사·자막 등). 무료 정책에 따라 **회의 시간·저장 횟수** 근접/초과 시 상단 또는 하단 배너. **AI 요약·AI 질의·클라우드 STT·공유** 등은 §2 에 따라 **제한 표시**(비활성 + 업그레이드 안내). |
| **`/app/record/:id`** | 기록 폼·메타 편집 **허용 범위 내** 표시. 저장 건수/첨부 등 한도 초과 시 저장 CTA 비활성 + 안내. AI 편집·공유가 유료 전용이면 동일하게 잠금. |

### 6.3 유료 회원 (`plan = pro`)

| 경로 | 화면에 보여야 하는 것 · 동작 |
|------|------------------------------|
| **`/app`** | 로그인 + **플랜 「Pro」(또는 유료)** 배지. 업셀 배너는 **없거나 최소**(예: 기관 기능 안내 정도). 나머지는 무료와 동일하게 진입 허브 안내 가능. |
| **`/app/meeting`** | 허브 전체 + **무료 대비 완화된 한도** 표시(선택). **AI·공유·긴 회의** 관련 메뉴 **활성**. 한도 여유가 있으면 경고 배너 생략 가능. |
| **`/app/room/:id`** | 회의실 **전 기능 노출**(§2 에서 Pro 에 허용한 범위): AI 탭·질의·고급 전사 등 **활성**. 여전히 상한이 있다면 상한 근접 시만 안내 배너. |
| **`/app/record/:id`** | 기록 편집 + **AI 보조·공유** 등 Pro 기능 **활성**. 조직 전용 필드가 없다면 무료·기관과 동일 폼에 기능만 확장. |

### 6.4 기관 회원 (`plan = org`)

| 경로 | 화면에 보여야 하는 것 · 동작 |
|------|------------------------------|
| **`/app`** | 로그인 + **플랜 「기관」**(또는 Org) 배지. 가능하면 **조직 이름** 한 줄 표시. 관리자 역할이면 **`[관리자]` 또는 관리 대시보드 진입**(후속 라우트와 연결). 일반 소속원은 동일 진입점에서 조직 배너만 차등 가능. |
| **`/app/meeting`** | 허브 + **조직 컨텍스트**(예: “○○ 기관 계정”). Pro 와 동등 이상의 한도 전제 시 기능 활성. 추가 메뉴(사용자 관리 등)는 **별도 경로**(예: `/app/org/...`) 도입 시 §6 보강. 본 문서에서는 허브 상단에 **기관 안내 배너** 정도로 충분. |
| **`/app/room/:id`** | 회의실은 Pro 와 동일 또는 상위 한도 UI + **보존·규정 안내 배너**(선택). 호스트/관리자에게만 보이는 초대·설정 항목은 역할 기반(후속). |
| **`/app/record/:id`** | 기록 편집 + **조직 보관·공유 규칙** 안내(예: 내부 공개 범위). 관리자에게만 삭제·이관 등 추가 버튼(후속 스키마와 연동). |

### 6.5 공통 UI 메모
